# Summary

In recent years, Python has become the most popular programming language in the scientific community. It offers extensive functionality for preparing, post-processing and visualising data of all types and origins. Through a large number of freely available packages, the programming environment can be flexibly adapted to a researcher's individual needs. Up until now fire simulation data as produced by the Fire Dynamics Simulator (FDS) [@fds] for example could not be easily imported and processed with Python. Although Smokeview [@smokeview] enabled the visualization of simulation results, it has limited functionality for conducting detailed quantitative analyses of the generated data. We publish the *FDSReader*, an open-source Python package to extract and analyze data generated by FDS simulations in a straightforward "Pythonic" and efficient way.

# Statement of need

At present, most working groups in both industry and research have no unified solution for efficiently extracting the binary data output of FDS in Python for further processing. This is a common requirement for both academic research and fire safety engineers who need to analyze data precisely and not rely on vague estimates via the GUI Smokeview provides.

# State of the field

Currently, there are two possible ways to analyze data generated by FDS: *Fds2Ascii*, a Fortran program to manually convert specific binary output files of FDS into a text-based format and subsequently read in the text file with the preferred programming language. Each FDS simulation outputs many of such binary files, sometimes hundreds or even thousands, depending on the size of the simulation. Converting all files is cumbersome, and interpreting the resulting text outputs remains non-trivial. Alternatively, researchers write custom scripts to read the binary data directly. Consequently, many groups maintain their own fragmented sets of (typically Python-based) scripts for various data types (e.g., slices, boundary data). Because the internal data structures of FDS are highly complex, these custom scripts are time-consuming to develop, often restricted to narrow simulation parameters (e.g., single-mesh only), and prone to breaking with new FDS releases. Therefore, rather than attempting to wrap the inefficient Fortran-based *Fds2Ascii* pipeline or maintain a fragmented ecosystem of limited custom scripts, developing a native, comprehensive, and actively maintained Python package from scratch was strictly necessary.

# Software design

The *FDSReader* Python package provides functionality to automatically import the full set of FDS outputs for both old and new versions of FDS. When first loading a simulation, all metadata is collected and cached to reduce subsequent load times. The complete data produced during the simulation - such as the slice data for each time step - is lazy-loaded, i.e., only fully loaded from disk when the data is accessed by the user via Python. The first access triggers loading the data into memory in the background and converting it to equivalent Python data structures. This method minimizes initial loading time without sacrificing the ability to easily filter and select the desired data. The *FDSReader* collects all data of the same type into one collection and offers its own set functions for each type to easily select the data which should be further processed. These functions can even be called in an automated fashion, to run some predefined post-processing routines on simulation data without having to manually interact with the data, as one would have to do when using SmokeView or Fds2Ascii. The *FDSReader* is able to read all data types available in FDS including most of the additional metadata. To be concrete, the package contains modules for slices (slcf), boundary data (bndf), volumes (plot3d, smoke3d) particles (part), isosurfaces (isof), evacuations (evac), complex geometry boundary data (geom), devices (devc) and meshes. Slices, boundary data, isosurfaces and volumes are each respectively separated into multiple parts, one for each mesh. While FDS outputs the data separately for each mesh, the *FDSReader* provides methods to combine all these parts automatically and operate on data across meshes.

The following code snippet demonstrates the use of *FDSReader* to load a temperature slice file from an FDS simulation. For the simulation times $t = 500\,\mathrm{s}$, the data are stored as a two-dimensional NumPy array. This array can, for example, be visualized using the `imshow` function of the `matplotlib.pyplot` module.

```python
import fdsreader as fds

sim = fds.Simulation('path/to/simulation')
slcf = sim.slices.filter_by_quantity('TEMPERATURE').get_by_id('SLCF_TEMP_1')
time_index = slcf.get_nearest_timestep(500)
data = slcf.to_global()[time_index]
```

# Research impact statement
Since the first release of the *FDSReader* in 2020, the package received widespread attention. According to ClickHouse download metrics [@clickhouse_metrics], the package has been downloaded over 263,000 times. It is supported by an active community, with issues and pull requests regularly submitted by both the original authors and external developers. Furthermore, the FDSReader ecosystem has spurred downstream development, leading to external forks and the creation of adjacent tools, such as an open-source web application interface [@fdsreader_webapp].
